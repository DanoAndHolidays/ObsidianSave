核心定义

|       |                                                                           |
| ----- | ------------------------------------------------------------------------- |
| 概念    | 说明                                                                        |
| CDN缓存 | CDN边缘节点（分布在全球的服务器）将源站内容临时存储在本地。用户请求时，若内容有效且存在，直接返回（“缓存命中”），实现毫秒级响应。       |
| 回源    | 当边缘节点无缓存、缓存过期或需验证内容时，自动向您的源站服务器（Origin Server）发起请求获取最新内容的过程。获取后通常会更新本地缓存。 |

---

 🔄 工作流程（用户访问资源时）

1. 用户请求资源（如 `logo.png`）
    
2. DNS将请求调度至**地理位置最近的****CDN****节点**
    
3. 节点检查缓存：
    
    1. ✅ **命中**：直接返回内容（高速、低延迟）
        
    2. ❌ **未命中/过期**：触发**回源** → 向源站拉取内容 → 缓存新内容 → 返回用户
        
4. 后续相同请求由该节点缓存直接响应
    

> 💡 _示例_：首次访问某新闻图片时CDN无缓存，回源拉取并缓存；1000名后续用户均从边缘节点秒开，源站仅承受1次压力。

---

 关键补充

- **缓存策略**
    
    - 由HTTP头（`Cache-Control`、`Expires`）或CDN后台规则控制（如TTL=24小时）
        
    - 可配置：哪些路径缓存（静态资源）、哪些不缓存（动态API）、缓存键（URL+参数等）
        
- **回源触发条件** 缓存缺失、TTL过期、手动刷新缓存、请求含`no-cache`头、缓存校验失败（如ETag变化）
    
- **回源优化**
    
    - 回源HOST/协议配置（HTTP/HTTPS）
        
    - 回源合并（多用户请求合并为1次回源，减轻源站压力）
        
    - 中间层节点（父层缓存）减少直达源站次数
