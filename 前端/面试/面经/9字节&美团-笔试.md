
### 1. 正则表达式标识题目

**题目**：请问以下关于正则表达式标识说法错误的是（）

- A. 现在的 RegExp 构造器禁止给已有任意标识的正则表达式传递任意标识。
- B. 通过正则表达式的 flags 属性获取的标识顺序，可能与正则表达式标识原有顺序不一致。
- C. 定点模式 y 用于重复匹配时，需要手动确保 lastIndex 指向正确的位置。
- D. 全局模式 g 在匹配过程中允许自由向前移动。

**参考答案**：A  
**解析**：

- **A选项**：ES6 中 RegExp 构造器允许给已有任意标识的正则表达式传递任意标识，传递的标识会覆盖原本的标识，选项错误。
- **B选项**：正则表达式的标识顺序都按照 gimuy，通过正则表达式的 flags 属性获取的标识顺序也遵守这个顺序，这可能与正则表达式标识原有顺序不一致，选项正确。
- **C选项**：定点模式 y 使用 lastIndex 作为精确且唯一的位置寻找匹配，不会向前移动去寻找匹配，因此当重复匹配时，需要手动确保 lastIndex 指向正确的位置，选项正确。
- **D选项**：全局模式 g 属于非定点匹配，因此在整个匹配过程中允许自由向前移动，选项正确。

### 2. CSS 选择器题目

**题目**：下面关于 CSS 选择器描述错误的是哪项？（）

- A. :checked表示任何已启用的元素
- B. :after用来创建一个伪元素，作为已选中元素的最后一个子元素
- C. :link表示尚未被访问的元素，匹配每个具有 href 属性的未访问的 或  元素
- D. :root匹配文档树的根元素

**参考答案**：A  
**解析**：

- **A选项**：错误，:checked 代表选择每个选中的输入元素，:enabled 选择每一个已启用的输入元素。
- **B选项**：正确，:after 伪元素在元素的内容之后插入内容。
- **C选项**：正确，:link 选择未访问过的链接。
- **D选项**：正确，:root 选择文档的根元素（HTML文档中通常是 html 元素）。

### 3. JS 代码输出题目

**题目**：请问以下 JS 代码最终输出的结果是（）

```javascript
class cls extends Array {
  func() {
    console.log(1);
  };
  static get [Symbol.species]() {
    return Array;
  }
};
const son = new cls(1, 2, 3);
const arr = son.filter((item) => item >= 2);
console.log(arr.constructor === Array);
arr.func();
```

- A. true、报错
- B. true、1
- C. false、1
- D. false、报错

**参考答案**：A  
**解析**：

- 类如果存在特殊的静态 getter 方法即 Symbol.species，那么内建方法如 map、filter 将使用它的返回值来创建新的结果数组。因此实例对象 son 通过 filter 方法创建的结果数组 arr 的 constructor 就是 Symbol.species 的返回值即 Array，所以 `arr.constructor === Array` 为 true。
- 由于数组 arr 的 constructor 是 Array 而非 cls，自然无法调用在类 cls 上拓展的新方法 func，强行调用将报错抛出异常。
- 综上，输出 true 和 报错。

### 4. 模块加载顺序题目

**题目**：请问以下模块加载顺序正确的是（）

```html
// 模块1:
<script type="module" src="./test.js"></script>
// 模块2:
<script type="module" src="./test.js" defer></script>
// 模块3:
<script src="./test.js" defer></script>
```

- A. 模块1、模块2、模块3
- B. 模块3、模块1、模块2
- C. 模块3、模块2、模块1
- D. 模块2、模块3、模块1

**参考答案**：A  
**解析**：

- 浏览器对于带有 `type="module"` 的 `<script>`，都是异步加载，不会造成堵塞浏览器，即等到整个页面渲染完，再执行模块脚本，等同于打开了 `<script>` 标签的 defer 属性。
- 也就是说模块1、模块2、模块3的加载行为都是等同一致的。
- 同时如果有多个 defer 脚本，会按照它们在页面出现的顺序加载，因此模块加载的顺序是模块1、模块2、模块3。

### 5. 直接插入排序算法题目

**题目**：用直接插入排序算法对下列4个表进行（从小到大）排序，比较次数相等的是（）

- A. [1, 3, 5, 2, 4]
- B. [3, 2, 5, 1, 4]
- C. [1, 2, 3, 4, 5]
- D. [5, 4, 3, 2, 1]

**参考答案**：AB  
**解析**：

- **对于 A**：总共需要 7 次比较。
- **对于 B**：总共需要 7 次比较。
- **对于 C**：总共需要 4 次比较（最好情况，基本有序）。
- **对于 D**：总共需要 10 次比较（最坏情况，逆序）。
- 故 A 和 B 的比较次数相等。

### 6. 二叉树遍历题目

**题目**：已知一棵二叉树的先序遍历结果为 ABCDEFG，则其可能的后序遍历结果为（）

- A. BFEGDCA
- B. CDEGFAB
- C. CBAEDGF
- D. CBFEGDA

**参考答案**：AD  
**解析**：

- 先序遍历的顺序是：根节点、左子树、右子树。
- 后序遍历的顺序是：左子树、右子树、根节点。
- 根据先序遍历结果 ABCDEFG，A 是根节点。
- **选项 A (BFEGDCA)** 和 **选项 D (CBFEGDA)** 的最后一个元素都是 A，符合后序遍历根节点在最后的特征，且子树结构逻辑成立。

### 7. WeakSet 实例创建题目

**题目**：下列创建WeakSet实例对象ws时，会抛出异常的是（）

- A. let vs = new WeakSet([1,2]);
- B. let vs = new WeakSet([{a:1}]);
- C. let vs = new WeakSet(1);
- D. let vs = new WeakSet({});

**参考答案**：ACD  
**解析**：

- WeakSet 的成员必须是对象，作为构造函数，WeakSet 接收一个具有 iterator 接口的结构作为参数（例如数组），该数组的成员自动成为 WeakSet 实例的成员。
- **A选项**：数组 [1,2] 作为参数，但是数组元素不是对象，会报错。
- **B选项**：数组 [{a:1}] 作为参数，且数组成员为对象，不会抛出异常。
- **C选项**：参数 1 没有实现 iterator 接口，且不是对象集合，会报错。
- **D选项**：参数 {} 没有实现 iterator 接口，WeakSet 构造函数需要可迭代对象，会报错。
- 综上，会抛出异常的是 A、C、D。
