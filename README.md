# Dano的知识库

这是我的个人知识库仓库，用来统一存放和整理所有的 Obsidian 笔记。

这里记录的不只是简单的学习摘抄，还包括日常开发中的知识沉淀、项目实践中的问题总结，以及对常见技术主题的系统化整理。相比把内容分散在不同位置，这个仓库更像一个可以持续积累、持续回顾、持续完善的个人知识空间。

## 知识库包含什么内容

当前知识库主要围绕技术学习和项目实践展开，内容覆盖多个方向，包括但不限于：

- 前端基础与进阶内容，比如 HTML、CSS、JavaScript、TypeScript、浏览器原理、工程化与性能优化。
- 常用框架与工具相关笔记，比如 Vue、React、Vite、Webpack、Git、Docker、npm 等。
- 后端与数据库相关内容，比如 Java、Express、SQL，以及部分开发环境与工具链整理。
- 项目笔记与问题记录，比如页面实现思路、功能开发过程、踩坑总结和业务场景分析。

## 这个知识库的作用

维护这个知识库的核心目的，是把零散的学习记录和实践经验整理成可复用、可检索、可迭代的资料。

很多内容在第一次学习时能够理解，但如果没有及时归档和反复整理，后续在真正使用时仍然容易遗忘。把笔记放在这里，可以帮助我在需要的时候更快找到答案，也能在持续补充的过程中逐步形成更清晰的知识结构。

这个知识库主要服务于以下场景：

- 学习某个知识点时，沉淀阶段性理解和重点结论。
- 做项目时，回查之前总结过的实现方案和经验。
- 复习时，快速定位某一类主题下的相关笔记。
- 长期积累时，把零散知识逐步整理成结构化内容。

## 组织方式

仓库中的内容按照主题和技术方向进行分类，例如前端、后端、公共技术、学习记录等。这样的组织方式方便按领域查找，也便于后续继续补充新的专题。

随着学习内容和项目经验不断增加，这个知识库也会持续更新、修订和重构，让已有内容更清晰，也更方便以后再次使用。
